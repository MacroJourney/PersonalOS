## 渐进式启动方案

在正式开始之前,你可以根据自己的时间和需求选择两种启动方式：

### 方式一：10分钟超快启动

**适合人群**：想快速体验，先建立概念

#### 第1步：创建文件夹（30秒）

```bash
# macOS/Linux
mkdir -p PersonalOS && cd PersonalOS

# Windows (PowerShell)
New-Item -Path "PersonalOS" -ItemType Directory -Force
cd PersonalOS
```

#### 第2步：复制唤醒咒语（1分钟）

```
🔮 进入数字分身模式

你现在是我的AI数字分身，不是普通助手。

第一个任务：引导我完成核心设定。
用对话方式问我5个问题，帮我写出一份1500字的"数字分身核心描述"。

包含：
1. 我是谁（身份、价值观）
2. 我的思维方式（如何看问题）
3. 我的做事风格
4. 我的沟通偏好
5. 我希望你如何协作

现在开始问第一个问题。
```

#### 第3步：跟着AI的问题回答（5分钟）

AI会逐个问你问题，你只需要真实回答。

#### 第4步：保存并测试（3分钟）

把AI生成的内容保存到文件，然后说：

```
"读取我的核心设定，用我的视角，
给我分析一下：我今天应该优先做什么事？"
```

**✅ 10分钟启动完成！**

你已经有了一个基础的数字分身，可以开始对话协作了。

---

### 方式二：30分钟完整启动

**适合人群**：想要完整体验，建立完整系统

#### 对比两种方式

| 维度 | 10分钟启动 | 30分钟启动 |
|-----|-----------|-----------|
| **文件结构** | 最小化 | 完整四区 |
| **核心配置** | 简化版 | 完整版 |
| **AI能力** | 对话协作 | 对话+记忆+复利 |
| **后续扩展** | 需要重建 | 直接使用 |
| **推荐场景** | 快速体验 | 长期使用 |

#### 选择建议

- 如果你只是想试试看 → 选10分钟启动
- 如果你确定要长期使用 → 选30分钟启动
- 如果不确定 → 先10分钟，满意后再升级到30分钟

## 阶段一：搭建基础（第1周）

### Day 1-2：创建目录结构

#### macOS/Linux 脚本

```bash
#!/bin/bash
# 创建 PersonalOS 完整目录结构

# 设置根目录（根据你的实际路径调整）
ROOT_DIR="$HOME/PersonalOS"

# 创建四区基础结构
mkdir -p "$ROOT_DIR"/{00_Core/memory,02_Human/{drafts,notes,personal,review_queue,archive},03_AI/{workspace/{drafts,experiments,temp},tools/{scripts,utilities},ai_diary,research,_private}}

# 如果有 Ideaverse Zero 2.5，复制为 01_Shared
# cp -r "/path/to/Ideaverse Zero 2.5" "$ROOT_DIR/01_Shared"

# 如果从零创建 01_Shared
mkdir -p "$ROOT_DIR/01_Shared"/{"+",Atlas/{Dots/{People,Things,Quotes,Statements,Questions,Skills},Maps,Sources/{Books,Articles,Courses}},Calendar/{Days,Records/{Meetings,Events,Ideas},Reviews/{Weekly,Monthly}},Efforts/{Areas,Projects/{Active,Simmering,Sleeping},Works},Strategy/{quarterly,reviews},Meta/{rules,ai-conversations},x/{Templates,"x/Prompts/contexts"}}

# 创建核心文件
touch "$ROOT_DIR/00_Core"/{profile.md,interaction_rules.md,stop_conditions.md,awakening_spell.md}
touch "$ROOT_DIR/00_Core/memory"/{long_term.md,recent_rolling.md,compression_rule.md}

echo "✅ PersonalOS 目录结构创建完成！"
echo "📂 位置：$ROOT_DIR"
echo "📝 下一步：用 Obsidian 打开 01_Shared/ 作为 Vault"
```

## 固定咒语

固定咒语是标准化的指令，帮助你快速启动常见流程，降低认知负担。

### 每天早晨（5分钟）

```
🌅 早晨启动咒语

请读取：
1. `00_Core/memory/recent_rolling.md`
2. `01_Shared/Efforts/Projects/Active/`
3. `01_Shared/Strategy/quarterly/当前季度.md`

今天建议我优先做什么？
- 给我3个选项（每个≤30分钟）
- 说明为什么选这个
- 与本周/本月目标的关联
- 预计完成时间和交付物
```

**AI会输出**：
- 今天建议做什么
- 为什么选这个（与目标的关联）
- 具体怎么做（3-7步行动方案）

**你只需要**：
- 回复"开始"
- 或者"改做XX"

### 任务执行中（按需使用）

#### 遇到困难时

```
遇到问题了：[简述问题]

要求：
1. 先查规则库和技能库，看是否有类似经验
2. 如果没有，搜索解决方案
3. 给我3个选项，说明优缺点
4. 推荐你认为最好的方案
```

#### 需要决策时

```
需要你帮我决策：[描述情况]

我的考虑因素：
1. [因素1]
2. [因素2]
3. [因素3]

请：
1. 分析每个选项的利弊
2. 基于我的价值观（profile.md）给建议
3. 说明你的推理过程
```

#### 需要暂停时

```
暂停当前任务。

请：
1. 保存当前进度
2. 记录到哪一步了
3. 下次继续需要什么信息
4. 更新记忆库
```

---

### 任务完成后（2分钟）

#### 结果满意

```
满意。请复盘并沉淀：
1. 写复盘到 `01_Shared/Calendar/Days/今日.md`
2. 提炼可复用资产（规则/技能）
3. 更新 `01_Shared/Atlas/Dots/Skills/_index.md`
4. 更新 `00_Core/memory/recent_rolling.md`
```

#### 结果不满意

```
不满意，[简述原因]。

要求：
1. 自己分析问题（不要问我）
2. 修复并重新交付
3. 把坑沉淀成规则，下次避免
4. 在规则中标注：⚠️ 易错点
```

#### 结果部分满意

```
70分，可以用，但有改进空间。

具体问题：
- [问题1]
- [问题2]

下次优化方向：
- [方向1]
- [方向2]

请记录到规则库，标注为"待优化"。
```
