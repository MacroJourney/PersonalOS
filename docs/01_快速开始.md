## 选择启动方式

根据你的时间和意图，选择一种启动方式：

| 维度 | 10 分钟超快启动 | 30 分钟完整启动 |
|-----|---------------|---------------|
| **你会得到什么** | 一份底层画像 + 基本对话能力 | 完整四区工作空间 + 配置文件 + 工具就绪 |
| **AI 能力** | 对话协作 | 对话 + 记忆 + 复利机制 |
| **后续扩展** | 需要补建结构 | 直接使用 |
| **适合人群** | 先体验再决定 | 确定要长期使用 |

**选择建议**：
- 不确定要不要用 → 先 10 分钟体验，满意后再升级
- 确定要长期用 → 直接 30 分钟完整启动
- 已经克隆了仓库 → 参考 [06_在仓库中搭建系统.md](06_在仓库中搭建系统.md)，在仓库内搭建

## 方式一：10 分钟超快启动

### 第 1 步：创建文件（30 秒）

```bash
mkdir -p PersonalOS/00_Core && cd PersonalOS
touch 00_Core/profile.md
```

### 第 2 步：用 AI 创建底层画像（1 分钟）

把以下内容发给 AI（Claude / ChatGPT / 任意 AI）：

```
请帮我创建 PersonalOS 的底层画像（profile.md）。

用对话方式逐个问我以下 5 个方向的问题，帮我写出一份 < 2000 字的底层画像：

1. 身份与背景：我是谁，做什么
2. 价值观与原则：我相信什么，怎么做决策
3. 思维方式与审美：我怎么思考，品味偏好
4. 驱动力与触发点：什么驱动我，什么让我兴奋
5. 性格与工作风格：个性特点，工作节奏

这份画像是我借给你的「灵魂」——让你以我的视角看世界、以我的标准做判断。
要求压缩到本质，不要写简历，不要写目标，只写稳定的「我是谁」。

现在开始问第一个问题。
```

### 第 3 步：跟着 AI 的问题回答（5 分钟）

AI 会逐个问你问题，真实回答即可。不需要刻意组织语言——AI 会帮你提炼。

### 第 4 步：保存并测试（3 分钟）

把 AI 生成的内容保存到 `00_Core/profile.md`，然后说：

```
读取我的底层画像（profile.md），用我的视角分析：
我今天应该优先做什么事？给我 3 个选项。
```

如果 AI 的回答让你感觉「它懂我」——恭喜，你的底层画像写对了。

**✅ 10 分钟启动完成！** 你已经有了系统的核心——底层画像。接下来可以直接用它对话协作，或升级到 30 分钟完整启动。

### 第 2 步：填写核心配置（15 分钟）

按优先级填写 `00_Core/` 的四个文件：

| 文件 | 回答的问题 | 怎么写 |
|-----|----------|-------|
| `profile.md` | **你是谁？** | 用 10 分钟启动中的方法，让 AI 引导你完成 |
| `interaction_rules.md` | AI 该怎么和你协作？ | 先写 3-5 条（沟通风格、格式偏好等），后续在使用中补充 |
| `stop_conditions.md` | 什么时候该停下来确认？ | 先写 3-5 个 AI 必须暂停请示的场景 |
| `awakening_spell.md` | 如何启动一次协作？ | 写一段标准启动指令，让 AI 加载 profile 和记忆 |

> 详细的模板和示例见 [03_模板库.md](03_模板库.md) 和 [examples/](../examples/)。

**最重要的是 `profile.md`**——它是整个系统的根基。其他文件可以先空着，在协作中逐步完善。

### 第 3 步：配置工具（10 分钟）

1. 用 **Obsidian** 打开 `01_Shared/` 作为 Vault
2. 安装 **Periodic Notes** 插件（自动创建每日/每周笔记）
3. 安装 **Templater** 插件（模板自动填充）
4. 把模板复制到 `01_Shared/x/Templates/`

> 工具配置详见核心设计文档 [六、工具配置](00_核心设计.md#六工具配置)。

**✅ 30 分钟启动完成！** 你的四区工作空间已经就绪。

## 渐进式采用路径

系统不需要一步到位。以下是建议的 4 周渐进路径：

### 第一周：养成记录习惯

**只做一件事**：每天在 `Calendar/Days/` 写一段记录。

- 不需要 AI，不需要复盘，不需要任何复杂流程
- 格式随意——今天做了什么、学了什么、想了什么
- 目标是养成「每天记一笔」的习惯

> 这一周的目的不是产出，是建立节奏。如果连记录都坚持不了，后面的循环也很难运转。

### 第二周：加入周复盘

- 周五花 15 分钟回顾本周的每日记录
- 可以手动写，也可以用下面的「周复盘咒语」让 AI 辅助
- 开始感受「回顾」带来的视角变化

### 第三周：加入知识沉淀

- 每晚花 5 分钟，从今日记录中提炼 1-2 个知识点
- 开始在 `Atlas/Dots/` 积累原子笔记
- 可以用下面的「晚间复盘咒语」让 AI 以 Skill Analyzer 模式辅助

### 第四周：完善循环

- 写下你的北极星（`Strategy/north-star.md`）和季度目标
- 月初尝试一次战略复盘（用「月度战略复盘咒语」）
- 三层 PDCA 循环开始运转

### 之后：持续迭代

- 不好用的环节可以暂时跳过，好用的环节加强
- 根据使用情况完善规则库和技能库
- 复利机制会随着积累逐渐显现效果

> 详细的 4 周计划见 [02_30天成长计划.md](02_30天成长计划.md)。

## 常用咒语

咒语是标准化的指令，帮你快速启动常见流程，降低认知负担。建议保存到 `01_Shared/x/Templates/Spells.md` 备用。

### 每天早晨（5 分钟）

```
🌅 早晨启动

请读取：
1. 00_Core/profile.md
2. 00_Core/memory/recent_rolling.md
3. 01_Shared/Efforts/Projects/Active/
4. 01_Shared/Strategy/quarterly/当前季度.md

今天建议我优先做什么？
- 给我 3 个选项（每个 ≤ 30 分钟）
- 说明与本周/本月目标的关联
- 推荐你认为最应该做的那个
```

---

### 任务执行中（按需）

#### 遇到困难

```
遇到问题：[简述问题]

要求：
1. 先查规则库（Meta/rules/）和技能库（Atlas/Dots/Skills/），看有无类似经验
2. 如果没有，搜索解决方案
3. 给我 3 个选项，说明优缺点
4. 推荐你认为最好的方案
```

#### 需要决策

```
需要你帮我决策：[描述情况]

我的考虑因素：
1. [因素1]
2. [因素2]
3. [因素3]

请：
1. 分析每个选项的利弊
2. 基于我的底层画像（profile.md）给建议
3. 说明你的推理过程
```

#### 需要暂停

```
暂停当前任务。

请：
1. 记录当前进度和上下文
2. 下次继续需要什么信息
3. 更新 00_Core/memory/recent_rolling.md
```

---

### 任务完成后（2 分钟）

#### 结果满意

```
满意。请复盘并沉淀：
1. 写复盘到 01_Shared/Calendar/Days/今日.md
2. 提炼可复用资产（规则/技能）
3. 更新技能索引 01_Shared/Atlas/Dots/Skills/_index.md
4. 更新记忆 00_Core/memory/recent_rolling.md
```

#### 结果不满意

```
不满意，[简述原因]。

要求：
1. 自己分析问题出在哪（不要问我）
2. 修复并重新交付
3. 把这次的坑沉淀成规则，避免下次再犯
4. 在规则中标注：⚠️ 易错点
```

#### 结果部分满意

```
70 分，可以用，但有改进空间。

具体问题：
- [问题1]
- [问题2]

下次优化方向：
- [方向1]
- [方向2]

请记录到规则库，标注为「待优化」。
```

### 每晚复盘（5 分钟）

```
🌙 晚间复盘（Skill Analyzer 模式）

分析今日记录（01_Shared/Calendar/Days/今日.md），告诉我：

1. 今日知识点提炼
   - 学到了什么新东西？有什么新洞察？

2. 建议沉淀到哪里
   - Atlas/Dots/[类别]/[文件名].md
   - 关联到哪些现有笔记或 MOC

3. 与现有知识的关联
   - 和之前哪些知识相关？可以更新哪个 MOC？

4. 明天建议
   - 基于今天的进展，明天应该优先做什么

然后更新 00_Core/memory/recent_rolling.md
```

### 每周五下午（15 分钟）

```
📊 周复盘（Project Manager 模式）

生成本周周报：

1. 读取本周每日记录（01_Shared/Calendar/Days/）
2. 读取 Active 项目（01_Shared/Efforts/Projects/Active/）
3. 读取季度目标（01_Shared/Strategy/quarterly/当前季度.md）

分析：
- 各项目进展如何？离目标多远？
- 哪些项目有风险或在拖延？为什么？
- 时间分配合理吗？高价值任务占比多少？
- 建议调整哪些项目状态（Active ↔ Simmering）？
- 下周 3 个优先事项是什么？

输出到：01_Shared/Calendar/Reviews/Weekly/YYYY-WXX.md
```

---

### 每月 1-3 日（30 分钟）

```
🎯 月度战略复盘（Strategy Coach 模式）

启动战略复盘：

1. 读取上月数据
   - 上月每日记录和周报
   - Active 项目状态
   - 季度目标和北极星

2. 战略质疑
   - 这些行动真的在接近目标吗？
   - 投入产出比最高的是哪 3 件事？
   - 哪些是伪工作？
   - 最大的机会成本是什么？
   - 如果只能做 1 件事，应该是什么？

3. 盲区识别
   - 我可能没看到的问题
   - 隐藏的机会成本
   - 需要调整的方向

4. 建议行动
   - 应该砍掉什么？加强什么？
   - 季度目标是否需要调整？

用苏格拉底式提问，引导我思考，不要直接给答案。

输出到：01_Shared/Strategy/reviews/YYYY-MM-review.md
```

### 特殊场景

#### 清理 Inbox

```
清理 Inbox（01_Shared/+/）：
1. 读取所有内容，先给分流建议（不要直接移动文件）
2. 按类别给建议路径：
   - 知识点 → Atlas/Dots/
   - 项目想法 → Efforts/Projects/Simmering/
   - 来源资料 → Atlas/Sources/Books/ 或 Atlas/Sources/Articles/ 或 Atlas/Sources/Clippings/
   - 含糊条目 → 进入待确认清单（给 2 个候选路径和理由）
3. 等我确认后，再由我执行或明确授权你执行具体移动
4. 输出标准版分流报告（6 字段）：
   - 总数
   - 建议分流数
   - 待确认数
   - 规则命中
   - 失败/不确定原因
   - 下一步建议
5. 任何删除建议都必须标注「需二次确认」，未经我明确确认不得删除
```

#### 启动新项目

```
启动新项目：[项目名称]

1. 在 01_Shared/Efforts/Projects/Active/ 创建项目文件
2. 确认该项目服务于哪个季度目标
3. 拆解为可执行的任务（每步 ≤ 15 分钟）
4. 识别风险点和所需资源
5. 设置里程碑（70 分标准 / 90 分标准）
```

#### 归档项目

```
归档项目：[项目名称]

1. 写项目复盘（目标达成、做对了什么、踩了什么坑）
2. 提炼可复用经验到规则库/技能库
3. 移动到 Sleeping（01_Shared/Efforts/Projects/Sleeping/）
4. 更新项目索引
```

#### 准备重要会议

```
帮我准备会议：[会议主题]

1. 读取相关项目文件
2. 整理关键信息和数据
3. 准备讨论要点（3-5 个）
4. 预判可能的问题和应对方案

输出：会议大纲 + 关键数据 + 讨论要点 + 决策选项
```

### 咒语使用技巧

1. **保存常用咒语**：在 `01_Shared/x/Templates/Spells.md` 集中管理，方便快速调用
2. **自定义咒语**：根据你的工作特点，创建专属场景咒语（如写作启动、学习复盘、决策分析）
3. **持续迭代**：用的过程中发现不够用就补充细节，发现多余就精简——让 AI 帮你优化咒语本身
