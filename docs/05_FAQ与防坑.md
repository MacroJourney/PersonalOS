## 常见问题

### Q1：01_Shared/ 和 02_Human/ 有什么区别？

|-----|--------|-----|
| 01_Shared/ | 可读取 | 协作区，AI 可以分析和建议 |
| 02_Human/ | 不进入 | 私人空间，私密想法、草稿 |

### Q2：三个 Claude 身份需要分开创建吗？

推荐分开创建独立 Projects，这样：
- 每个身份有专门的 System Prompt
- 上下文更聚焦
- 避免角色混淆

简化方案：1个 Project，通过对话指令切换角色。

### Q3：这个系统会不会太复杂？

关键是**渐进式采用**：
1. 第一阶段：只用 00_Core + Calendar + Strategy
2. 第二阶段：加入 Projects + 周复盘
3. 第三阶段：完善 Atlas + 三个AI身份

### Q4：不会编程可以用吗？

完全可以。核心操作：
- 用 Obsidian 写 Markdown 笔记
- 用 Claude 对话（网页版即可）
- 不需要任何代码

### Q5：安全边界怎么设置？

三个等级：
- 🟢 **最安全**：单独一台电脑/虚拟机给AI用
- 🟡 **折中**：单独一个系统账户
- 🟠 **最低要求**：只给AI一个专用文件夹权限

**绝对不要放进系统的东西**：
- 身份证、银行卡号、密码
- 公司机密文档
- API Key（如必须用，放 .env 文件）

### Q10：会不会"失控"？（4层控制机制）

**完全不会，系统内置4层控制**：

**第1层：文件权限控制**
```
AI只能访问：
✅ PersonalOS/ 文件夹
❌ 其他盘/文件夹完全隔离

实现方式：
- 方案A：单独电脑/虚拟机
- 方案B：单独系统账户
- 方案C：只授权一个文件夹
```

**第2层：安全边界规则**
```
写在 00_Core/interaction_rules.md：

未经确认，AI不得：
❌ 付费/下单/购买
❌ 对外发布内容
❌ 删除或覆盖原文件
❌ 访问 02_Human/ 私人空间

违反 = 严厉批评 + 强制更新规则
```

**第3层：停止条件机制**
```
写在 00_Core/stop_conditions.md：

自动停止条件：
- 单任务最多3轮尝试
- 成本超过预设阈值
- 连续失败无改进
- 收益递减明显

防止无限循环和资源浪费
```

**第4层：人类最终决策权**
```
重大决策AI会询问：
- 需要较大消耗（>30分钟、>¥50）
- 涉及战略调整
- 不确定的选择

最终决定权永远在你
```

**实际体验**：
> "我现在很多时候不知道AI在做什么，
> 但所有结果都是可控的，而且经常给我惊喜。
> 这种'不知道过程但结果可控'，
> 就像雇了个靠谱的员工。"

### Q11：如何防止AI"偷懒"或应付？（复利工程机制）

**防偷懒的5个机制**：

**机制1：强制复盘**
```
每个任务必须写复盘：
- 没写 = 任务未完成
- 复盘质量 = AI的成长速度

检查点：
- 做对了什么？
- 踩了什么坑？
- 下次怎么更好？
```

**机制2：随机抽查**
```
定期问AI：
"这次比上次好在哪？"
"你用了哪条规则？"
"为什么选这个方案？"

答不上来 = 没有真正学习
```

**机制3：自我测试**
```
AI完成后必须自己检查：
- 核心功能可用吗？
- 达到70分标准了吗？
- 有没有遗漏的风险？

不要等你发现问题
```

**机制4：真实记录**
```
失败也要记录：
- 尝试了什么方法？
- 为什么没成功？
- 学到了什么？

失败是宝贵的经验
```

**机制5：规则库验证**
```
定期检查：
- 规则库是否在增长？
- 新规则是否被使用？
- 旧规则是否被优化？

规则库 = AI成长的证据
```

**实际效果**：
> AI会说"我穷尽了所有方法"，
> 并详细列出尝试过程：
> 1. 先查了规则库，找到XX规则
> 2. 尝试了方案A，失败原因是YY
> 3. 搜索了ZZ，找到替代方案B
> 4. 最终成功，已沉淀为新规则
>
> 这证明它真的在努力。

### Q13：隐私安全怎么保护？（分级管理）

**3级分类，分别处理**：

```markdown
🟢 Level 1 - 公开信息
可以用在线AI：
- 学习笔记
- 工作想法
- 公开资料
- 技术问题

风险：几乎无
处理：正常使用

---

🟡 Level 2 - 个人信息
用可信服务（Claude/ChatGPT）或本地AI：
- 日记（非敏感部分）
- 个人计划
- 项目文档
- 工作记录

风险：低
处理：
- 用大厂服务（有隐私政策）
- 或用本地AI（如Ollama）
- 定期删除对话记录

---

🔴 Level 3 - 绝对隐私
不输入任何AI：
- 密码、验证码
- 身份证、银行卡号
- 财务信息
- 公司机密
- 他人隐私

风险：高
处理：完全不用AI
```

**5个保护技巧**：

**技巧1：用代号**
```
真实信息 → 代号
- 公司名 → "A公司"
- 真实姓名 → "张某"
- 项目名 → "X项目"
```

**技巧2：定期检查**
```
每周检查AI工作区：
- 有没有敏感信息？
- 需要删除什么？
- 规则库是否泄露隐私？
```

**技巧3：脱敏规则**
```
写入 00_Core/interaction_rules.md：

涉及隐私时：
⚠️ 先提醒我脱敏
⚠️ 不在日志中记录敏感原文
⚠️ 用代号替代真实信息
```

**技巧4：本地优先**
```
Level 3信息：
- 用本地AI（Ollama + Llama）
- 或者完全不用AI
- 人工处理
```

**技巧5：对话删除**
```
重要对话后：
- 手动删除对话记录
- 只保留沉淀的规则/技能
- 规则中不包含敏感信息
```

| 任务类型 | AI擅长度 | 建议分工 | 预期效果 | 注意事项 |
|:---|:---:|:---|:---:|:---|

| **内容创作** | ⭐⭐⭐⭐⭐ | AI 80%，你20% | 效率提升5倍 | 保持你的风格 |
| **编程开发** | ⭐⭐⭐⭐⭐ | AI写，你验收 | 0基础也能做 | 理解核心逻辑 |
| **数据分析** | ⭐⭐⭐⭐ | AI处理，你解读 | 节省大量时间 | 验证数据来源 |
| **项目管理** | ⭐⭐⭐⭐ | AI跟踪，你决策 | 不遗漏细节 | 保持主导权 |
| **学习总结** | ⭐⭐⭐⭐ | AI提炼，你吸收 | 加速理解 | 主动思考 |
| **创意构思** | ⭐⭐⭐ | AI提供10个，你选1个 | 拓展思路 | 保持独特性 |
| **战略决策** | ⭐⭐ | AI分析，你决策 | 辅助参考 | 不能完全依赖 |

| **情感支持** | ⭐ | 谨慎使用 | 有限帮助 | 不能替代真人 |

**判断原则**：

✅ **完全交给AI**：
- 规律性强（有明确步骤）
- 标准明确（可量化评估）
- 可重复（做过类似的）
- 低风险（出错影响小）

🤝 **AI辅助，你决策**：
- 需要判断（多个选项）
- 涉及人际（沟通协调）
- 战略层面（长期影响）
- 创新性（需要独特视角）

⚠️ **谨慎使用AI**：
- 需要共情（情感理解）
- 涉及隐私（敏感信息）
- 法律合规（需要专业判断）
- 伦理道德（价值观相关）

**实战建议**：
1. 从最简单的任务开始（信息整理）
2. 逐步扩展到复杂任务（内容创作）
3. 始终保持人类的最终决策权
4. 定期评估AI的表现和适用范围

### Q15：如何知道AI是否真的在成长？（量化指标）

**4个量化指标**：

| 指标 | 衡量方法 | 目标 | 检查频率 |
|:---|:---|:---:|:---:|
| **完成速度** | 同类任务耗时 | 每月减少20% | 每周 |
| **通过率** | 一次通过的比例 | 第1月60%→第3月80% | 每周 |
| **规则数** | 规则库文件数量 | 第1月10条→第3月30条 | 每月 |
| **主动性** | AI主动发现问题次数 | 从0到每周3-5次 | 每周 |

**每周测试法**：

```
Week 1：让AI总结一篇文章
        → 记录：耗时20分钟，质量70分

Week 2：同类文章
        → 对比：耗时15分钟，质量75分

Week 4：同类文章
        → 期待：耗时10分钟，质量85分

如果没进步 → 检查复利工程是否运行
```

**成长检查清单**：

```markdown
## 每周检查（5分钟）

### 速度指标
- [ ] 同类任务是否更快？
- [ ] 是否减少了重复询问？
- [ ] 是否主动调用规则/技能？

### 质量指标
- [ ] 一次通过率是否提升？
- [ ] 是否减少了低级错误？
- [ ] 是否能预判我的需求？

### 成长指标
- [ ] 规则库是否增加？
- [ ] 新规则是否被使用？
- [ ] 是否有主动优化？

### 主动性指标
- [ ] 是否主动发现问题？
- [ ] 是否主动提出建议？
- [ ] 是否主动探索新方法？
```

**如果没有成长，检查**：

1. **复盘是否执行？**
   - 每次任务后是否写复盘？
   - 复盘质量如何？

2. **规则是否沉淀？**
   - 可复用经验是否写成规则？
   - 规则是否被AI读取和使用？

3. **反馈是否明确？**
   - 你的标准是否清晰？
   - AI是否理解你的期待？

4. **系统是否运转？**
   - 记忆是否更新？
   - 文件结构是否合理？

**实际案例**：

```
第1个月：
- 完成速度：20分钟/任务
- 通过率：60%
- 规则数：8条
- 主动性：0次

第3个月：
- 完成速度：8分钟/任务（提升60%）
- 通过率：85%（提升42%）
- 规则数：35条（增长337%）
- 主动性：每周4-6次

结论：系统运转良好，AI在持续成长
```

## 防坑指南

### 坑1：把复盘变成资料收集黑洞

**症状**：花3天收集资料，整理笔记，越做越累，没力气真正复盘

**解法**：
```
❌ 不要：先收集→再复盘
✅ 应该：先问why/what→只收集必要信息

步骤：
1. 先问：我为什么复盘？想得到什么？
2. 列出：需要哪些信息才能回答？
3. 只收集：能改变决策的关键信息
4. 停止：收集够了就停，别追求完美
```

---

### 坑2：套用别人的模板，越写越不是你

**症状**：用OKR模板复盘，感觉很僵硬，最后都放弃了

**解法**：
```
在 profile.md 写清楚你的判断标准：
- 你如何评价"好"和"坏"
- 你的决策优先级
- 你的思考方式

AI会用"你的方式"而不是"标准模板"
```

---

### 坑3：想写给别人看，越写越端着

**症状**：想着要发朋友圈，越写越正式，根本不是为自己写的

**解法**：
```
02_Human/ 就是你的私人空间：
- 先为自己写（不考虑发布）
- 写完觉得值得分享，再提炼到 01_Shared/Works/
- AI帮你把私人笔记转成公开内容
```

---

### 坑4：AI太主动，人脑过载

**症状**：AI一小时做20件事，你根本看不过来

**解法**：
```
在 interaction_rules.md 中设置：
- 默认不实时打断
- 只在固定时间汇报
- 紧急情况才即时通知

过载保护：
- 如果连续说"随便"，AI自动放慢节奏
```

---

### 坑5：参与太多，效果反而变差

**症状**：你参与度70%，结果比AI全自动还差

**解法**：
```
最佳参与度通常是 10%-20%：
1. 先让AI 100%自主完成
2. 看下限在哪
3. 再逐步加入你的判断
4. 找到最佳平衡点
```
