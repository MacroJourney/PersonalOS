# 三层 PDCA 系统

> 版本：v2.1  
> 创建日期：2026-02-02  
> 文档类型：架构设计

## 概述

三层 PDCA 系统是 PersonalOS 的核心方法论，通过三个嵌套的反馈循环实现持续改进。

## 三个层次

### 战略层 PDCA（季度/年度）

**目标**：确保长期方向正确

**周期**：季度复盘

**内容**：
- **Plan**：设定年度/季度 OKR
- **Do**：执行季度计划
- **Check**：季度复盘，检查目标达成情况
- **Act**：调整战略方向

**工具**：
- `01_Shared/Strategy/vision.md` - 愿景
- `01_Shared/Strategy/quarterly-okr.md` - 季度 OKR
- `01_Shared/Calendar/Reviews/` - 季度复盘

### 项目层 PDCA（周/月）

**目标**：确保项目按计划推进

**周期**：周复盘

**内容**：
- **Plan**：制定项目计划和里程碑
- **Do**：执行项目任务
- **Check**：周复盘，检查进度
- **Act**：调整项目计划

**工具**：
- `01_Shared/Efforts/Projects/` - 项目管理
- `01_Shared/Calendar/Reviews/` - 周复盘
- `01_Shared/Meta/rules/` - 项目规则

### 能力层 PDCA（每日）

**目标**：持续积累可复用资产

**周期**：每日复盘

**内容**：
- **Plan**：每日计划
- **Do**：执行任务
- **Check**：每日复盘
- **Act**：提炼规则、技能、模板

**工具**：
- `01_Shared/Calendar/Days/` - 每日笔记
- `01_Shared/Meta/rules/` - 决策规则库
- `01_Shared/Atlas/` - 知识库

## 复利增长机制

### 规则库

每次决策后，提炼可复用的决策规则：

```markdown
# 规则：如何选择技术栈

## 触发条件
- 开始新项目时
- 需要选择技术方案时

## 决策标准
1. 团队熟悉度 > 技术先进性
2. 生态成熟度 > 新鲜感
3. 长期维护成本 > 短期开发速度

## 历史案例
- 2026-01-15：选择 React 而非 Vue（团队更熟悉）
- 2026-01-20：选择 PostgreSQL 而非 MongoDB（生态更成熟）
```

### 技能库

每次学习后，记录可复用的技能：

```markdown
# 技能：如何使用 Claude Projects

## 适用场景
- 需要 AI 长期记住上下文
- 多次对话涉及同一项目

## 操作步骤
1. 创建 Project
2. 上传核心文档（profile.md、rules.md）
3. 在对话中引用 Project

## 注意事项
- 文档不要超过 200KB
- 定期更新 Project 内容
```

### 模板库

每次完成任务后，提炼可复用的模板：

```markdown
# 模板：项目启动文档

## 项目概述
- 项目名称：
- 项目目标：
- 成功标准：

## 关键里程碑
- [ ] 里程碑 1
- [ ] 里程碑 2

## 风险和假设
- 风险 1：
- 假设 1：
```

## 三层联动

### 自下而上

- **能力层** → **项目层**：每日积累的规则和技能，支撑项目执行
- **项目层** → **战略层**：项目成果和经验，验证战略方向

### 自上而下

- **战略层** → **项目层**：战略目标分解为具体项目
- **项目层** → **能力层**：项目需求驱动能力建设

## 实施指南

### 每日流程

1. **早晨**（Plan）：
   - 查看今日计划
   - 确定优先级

2. **工作中**（Do）：
   - 执行任务
   - 记录过程

3. **晚上**（Check + Act）：
   - 每日复盘
   - 提炼规则/技能/模板

### 每周流程

1. **周一**（Plan）：
   - 制定周计划
   - 对齐项目目标

2. **周中**（Do）：
   - 推进项目
   - 调整计划

3. **周五**（Check + Act）：
   - 周复盘
   - 更新项目状态

### 每季度流程

1. **季初**（Plan）：
   - 设定季度 OKR
   - 分解为项目

2. **季中**（Do）：
   - 执行项目
   - 月度检查

3. **季末**（Check + Act）：
   - 季度复盘
   - 调整战略
