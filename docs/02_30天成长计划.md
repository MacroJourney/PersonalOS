## Week 1：建立基础（每天1-2小时）

### Day 1-2：搭建系统

**任务清单**：
- [ ] 选择启动方式（10分钟 or 30分钟）
- [ ] 创建文件夹结构
- [ ] 写核心描述（可以用AI帮助）
- [ ] 创建唤醒咒语

**完成标准**：
- ✅ 文件夹结构完整
- ✅ 核心文件都有内容（哪怕粗糙）

**常见挑战**：
- 不知道如何描述自己 → 用AI的5个问题引导
- 担心写得不够好 → 先写70分版本，后续可以迭代

---

### Day 3-4：首次协作

**任务清单**：
- [ ] 第一次唤醒AI，让它理解你
- [ ] 给3个简单任务测试：
  1. 搜索资料并总结
  2. 整理你的笔记
  3. 生成一个待办清单
- [ ] 检查AI是否写了日志
- [ ] 给第一次反馈

**完成标准**：
- ✅ AI能用你的风格表达
- ✅ 日志文件夹有记录
- ✅ 你理解了基本协作流程

**常见挑战**：
- AI的回复太"AI味" → 在profile.md中明确你的表达风格
- 不知道给什么任务 → 从日常工作中选最简单的开始

---

### Day 5-7：建立习惯

**每天流程**：

```
早晨（5分钟）：
- 用唤醒咒语启动
- AI建议今天的任务

白天（30分钟）：
- 给至少1个任务
- 让AI自己完成

晚上（5分钟）：
- 验收结果
- 给反馈
- AI更新记忆
```

**完成标准**：
- ✅ 完成至少5个任务
- ✅ 规则库有3-5条规则
- ✅ 形成固定的协作节奏

**常见挑战**：
- 忘记每天使用 → 设置手机提醒
- 任务完成质量不稳定 → 明确你的评分标准

---

## Week 2：提升质量（每天30-60分钟）

### Day 8-10：优化反馈机制

**练习任务**：

```
Day 8：
给任务 → AI完成 → 你只说"60分，因为XX"
→ 让AI自己优化

Day 9：
同类任务 → 看AI是否记住昨天的教训

Day 10：
如果AI记住了 = 成功！
如果没有 = 检查规则库是否更新
```

**完成标准**：
- ✅ 学会给"标准"而非"细节"
- ✅ AI能自己提炼规则
- ✅ 任务通过率 > 60%

**常见挑战**：
- 不知道如何给反馈 → 用"好在哪/差在哪/如何改进"三段式
- AI没有记住教训 → 检查是否写入了规则库

---

### Day 11-14：扩展应用

**选择一个方向深入**：

**Option A：内容创作**
- [ ] 写一篇文章或短视频脚本
- [ ] 定义你的风格规则
- [ ] 让AI生成3个版本对比

**Option B：知识管理**
- [ ] 让AI整理你的笔记
- [ ] 建立知识索引（MOC）
- [ ] 测试知识检索效率

**Option C：项目管理**
- [ ] 创建一个项目
- [ ] 让AI帮你拆解任务
- [ ] 每天更新进度

**完成标准**：
- ✅ 做出第一个成果（内容/知识库/项目）
- ✅ 规则库达到8-10条
- ✅ 感受到效率提升

**常见挑战**：
- 不知道选哪个方向 → 选你当前最需要的
- 成果不满意 → 降低期待，先做到70分

---

## Week 3：自主工作（每天20-40分钟）

### Day 15-17：部分放权

**测试流程**：

```
早上（5分钟）：
给3个灵感，让AI选一个自己做

中午：
不管它

下午（10分钟）：
检查结果，给反馈

晚上（5分钟）：
听AI汇报，更新记忆
```

**完成标准**：
- ✅ AI能独立完成70%的任务
- ✅ 你的介入时间减少到20分钟/天
- ✅ 形成"主动但不打扰"的节奏

**常见挑战**：
- 不放心让AI自主 → 从最简单的任务开始放权
- AI做的不满意 → 检查是否给了明确的标准

---

### Day 18-21：完全自主（某个领域）

**选择一个领域完全交给AI**：

- 每日新闻整理
- 学习笔记总结
- 项目进度跟踪
- 灵感库维护

**测试标准**：

```
Day 18：你告诉AI规则和标准
Day 19-21：AI自己做，你只定期检查（每天5分钟）

如果连续3天都满意 = 成功！
```

**完成标准**：
- ✅ 某个领域AI可以100%自主
- ✅ 规则库达到15条
- ✅ 你的时间节省50%以上

**常见挑战**：
- AI偷懒或应付 → 设置自我测试机制
- 质量不稳定 → 完善规则库和停止条件

---

## Week 4：深化协作（每天30-60分钟）

### Day 22-25：探索边界

**挑战任务**：

让AI做之前没做过的事：

**任务1：分析你的时间使用**
```
读取我这个月的日志，分析：
1. 我主要时间花在哪里？
2. 哪些任务可以优化/委托？
3. 给我下个月的时间分配建议
```

**任务2：设计一个新产品**
```
基于我的兴趣和能力，
设计一个我可以做的小产品：
- 解决什么问题？
- 目标用户是谁？
- 最小可行版本（MVP）是什么？
```

**任务3：给你的目标提建议**
```
读取我的长期目标，
基于我这个月的行动，
分析：
- 我是否在朝目标前进？
- 哪些行动是有效的？
- 下个月应该调整什么？
```

**完成标准**：
- ✅ AI能主动给你提供价值
- ✅ 不只是执行，还能建议
- ✅ 你感受到AI是"合伙人"而非"工具"

---

### Day 26-28：优化系统

**回顾与优化**：

```
任务：和AI一起优化系统

1. 回顾30天的日志
   - 哪些规则最常用？
   - 哪些规则从未用过？

2. 优化规则库
   - 合并相似规则
   - 删除无用规则
   - 重要规则排序

3. 优化文件结构
   - 是否需要新的分类？
   - 文件命名是否清晰？

4. 规划下个月
   - 继续深入哪个方向？
   - 尝试什么新应用？
```

**完成标准**：
- ✅ 规则库精简优化
- ✅ 系统更适合你的节奏
- ✅ 有明确的下月目标

---

任务：和AI一起写成长报告

对比 Day 1 和 Day 30：

【系统层面】
- 文件数量：[Day 1] → [Day 30]
- 规则数量：[0] → [XX]

【能力层面】

【感受层面】

**庆祝方式**：

- 或者只给自己，记录这个里程碑
- 规划第二个月的旅程

## 效果预期

### 第1个月：建立习惯

**目标**：
- 形成稳定的协作节奏
- 建立基础的规则库和技能库
- AI能理解你的基本风格和标准

**挑战**：
- 不习惯每天使用
- 不知道如何给反馈
- 对AI的能力期待过高或过低

**解决方案**：
- 设置固定提醒
- 使用固定咒语降低认知负担
- 从简单任务开始，逐步提升难度

---

### 第2-3个月：形成惯性

**目标**：
- AI能自主完成大部分常规任务
- 规则库和技能库持续增长
- 你的介入时间大幅减少

**效果**：
- 任务完成速度提升50-80%
- 任务一次通过率达到70-80%
- 每天只需20-30分钟管理AI

---

### 第4个月起：系统内化

**目标**：
- AI成为你思考和工作的延伸
- 系统自动运转，你只做战略决策
- 持续产出高质量成果

**效果**：
- AI能预判你的需求
- 主动发现问题和机会
- 你的生产力提升3-5倍
