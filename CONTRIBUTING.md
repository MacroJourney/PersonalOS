# 贡献指南

感谢你对 PersonalOS 的关注！

## 如何贡献

虽然本项目主要用于个人迭代需求，但我们欢迎以下类型的贡献：

### 1. 报告问题

如果你发现了问题或有改进建议：

1. 访问 [Issues](https://github.com/MacroJourney/PersonalOS/issues)
2. 点击 "New Issue"
3. 清晰描述问题或建议
4. 如果可能，提供截图或示例

### 2. 改进文档

文档改进总是受欢迎的：

1. Fork 本仓库
2. 创建你的分支 (`git checkout -b docs/improve-readme`)
3. 修改文档
4. 提交你的修改 (`git commit -m 'docs: improve README clarity'`)
5. 推送到分支 (`git push origin docs/improve-readme`)
6. 开启一个 Pull Request

### 3. 分享使用案例

如果你使用 PersonalOS 并有心得：

1. 在 Issues 中分享你的使用案例
2. 或者在 Discussions 中讨论
3. 我们可能会将优秀案例添加到 `examples/` 目录

### 4. 提出新功能

如果你有新功能建议：

1. 先在 Issues 中讨论
2. 说明功能的用途和价值
3. 如果可能，提供设计草图或伪代码

## 贡献类型

欢迎的贡献：
- ✅ 文档改进（错别字、表述优化、补充说明）
- ✅ Bug 修复
- ✅ 使用案例分享
- ✅ 模板优化
- ✅ 翻译（如果需要英文版）

不接受的贡献：
- ❌ 与 PersonalOS 核心理念不符的功能
- ❌ 过度复杂化的设计
- ❌ 未经讨论的大规模重构

## 代码规范

### 文档规范

- 使用简体中文
- 保持简洁清晰
- 使用 Markdown 格式
- 遵循现有的文档结构

### 提交信息规范

使用语义化提交信息：

```
类型: 简短描述

详细描述（可选）
```

类型：
- `docs`: 文档修改
- `feat`: 新功能
- `fix`: Bug 修复
- `refactor`: 重构
- `style`: 格式调整
- `chore`: 其他修改

示例：
```
docs: 更新快速开始指南

- 添加 SSH 配置说明
- 补充常见问题解答
```

## 行为准则

- 尊重他人
- 保持友善和专业
- 接受建设性批评
- 关注项目目标
