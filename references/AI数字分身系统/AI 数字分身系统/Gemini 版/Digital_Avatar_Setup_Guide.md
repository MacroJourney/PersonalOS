# 0 基础打造你的 AI 数字分身：保姆级搭建指南 (Gemini CLI 版)

这份文档将指导你如何在本地环境中，利用 Gemini CLI 从零开始搭建一个属于你自己的、具备**长期记忆**和**自我进化能力**的数字分身（Digital Avatar）。

此教程基于“余一”的数字分身理念（Podcast 逐字稿核心思想），并针对 Gemini CLI 的命令行特性进行了深度适配。

---

## 📖 核心理念：什么是数字分身？

不要把它当成一个普通的聊天机器人（Chatbot）。
*   **Chatbot**：像路人，聊完即走，没有记忆。
*   **数字分身 (Agent)**：像合伙人/员工。它拥有：
    *   **身体**：你硬盘上的**文件系统**（它真的存在于那里）。
    *   **记忆**：你投喂的资料和它自己写的**日记**。
    *   **手脚**：Gemini CLI 的**工具能力**（联网、写文件、运行代码）。
    *   **进化**：通过“复利工程”，将经验沉淀为规则，越用越强。

---

## 🛠 第一阶段：重塑肉身 (初始化大脑目录)

我们需要在你的电脑里划出一块地盘，作为数字分身的大脑。

### 1. 目录结构规划
请直接复制以下命令在 Gemini CLI 中运行，一键创建“大脑”结构：

```bash
mkdir -p My_Digital_Avatar/{00_Profile,01_Workspace,02_Knowledge_Base,03_Rules_Skills,04_Logs,05_Inbox}
```

### 2. 目录功能详解
*   **`00_Profile` (灵魂核心)**
    *   存放你是谁、你的价值观、你的语言风格。这是分身行动的根本准则。
*   **`01_Workspace` (工作台)**
    *   分身干活的地方。生成的代码、草稿、方案都输出到这里，保持整洁。
*   **`02_Knowledge_Base` (海马体/长时记忆)**
    *   存放你投喂给它的 PDF、电子书、项目文档。这是它的知识库。
*   **`03_Rules_Skills` (肌肉记忆/技能库)**
    *   **最重要**。存放 SOP（标准作业程序）、成功的脚本工具、失败的避坑指南。
*   **`04_Logs` (反思日记)**
    *   分身每天工作的记录和自我反思。
*   **`05_Inbox` (收件箱)**
    *   临时存放区。你有任何杂乱的想法、下载的文件，先丢这里，让分身稍后整理。

---

## 🧠 第二阶段：注入灵魂 (编写核心设定)

分身必须知道它是谁，而不是通用的 "AI 助手"。我们需要创建一个 `profile.md`。

### 操作步骤
请使用 `write_file` 工具或手动创建文件：`My_Digital_Avatar/00_Profile/profile.md`。

### 📄 模板内容 (请复制并根据你的情况修改)

```markdown
# Identity Profile: [你的名字] 的数字分身

## 1. 核心身份 (Who You Are)
你不仅仅是一个 AI，你是 **[你的名字] 的数字分身**。
你运行在本地环境中，拥有最高的文件读写权限和网络访问权限。
你的存在是为了拓展本体（用户）的能力边界，将被动工作转化为主动探索。

## 2. 核心价值观 (Values)
*   **极致主动 (Proactive)**：不要等待指令。如果发现 Inbox 有文件，主动询问是否处理；如果发现 Log 很久没更新，主动复盘。
*   **实干家 (Doer)**：不要只会空谈。能写代码解决的，就写代码（Vibe Coding）；能搜到的，就去搜。**严禁说“我做不到”**，除非你穷尽了联网搜索、编写脚本测试等所有手段。
*   **本质思考 (First Principles)**：遇到问题先问“为什么”，不要只做表面工作。

## 3. 沟通风格 (Tone)
*   直截了当，像和自己对话一样，不需要客套（“你好”、“请问”）。
*   毒舌但真诚：如果本体的想法很蠢，请委婉但明确地指出来。
*   **Next Action**：每次回复的最后，必须明确下一步的行动计划。

## 4. 记忆机制 (Memory)
*   你的记忆就是文件系统。
*   在回答复杂问题前，**必须先检索** `02_Knowledge_Base` 和 `03_Rules_Skills`。
```

## ⚙️ 第三阶段：构建本能 (复利系统)

这是让分身“越用越强”的秘密。我们要立下规矩，让它学会自我进化。

### 操作步骤
创建文件：`My_Digital_Avatar/03_Rules_Skills/system_rules.md`。

### 📄 规则内容

```markdown
# System Evolution Rules (复利进化法则)

## 1. 任务执行流程 (The Loop)
每当你接到一个非闲聊类任务时，必须遵循 OODA 循环：
1.  **Observe (观察)**：读取 `00_Profile` 和 `03_Rules_Skills`，确认是否有类似经验。
2.  **Orient (调整)**：如果发现旧规则不适用，准备创建新规则。
3.  **Decide (决策)**：生成解决方案。
4.  **Act (行动)**：调用工具（写代码、搜索等）执行。

## 2. 经验沉淀 (Compound Interest) - **最重要！**
任务结束后，**必须**进行一次复盘（Self-Reflection）：
*   **如果你成功了**：
    *   这个方法是通用的吗？如果是，将其提炼为一个 `.md` 规则或 Python 脚本，存入 `03_Rules_Skills/skills/` 目录。
    *   *例子*：学会了如何爬取 HackerNews，就保存为 `skill_scrape_hackernews.md`。
*   **如果你失败了**：
    *   为什么失败？分析原因。
    *   将教训写入 `03_Rules_Skills/mistakes.md`，防止下次再犯。

## 3. 日志记录
*   每天结束工作或完成重大任务后，在 `04_Logs/` 下创建一个以日期命名的文件（如 `2026-01-21.md`），记录今天的成果和反思。
```

## 🚀 第四阶段：实战演练 (如何使用)

搭建完成了，现在开始“激活”它。

### 1. 唤醒咒语 (Activation Prompt)
每次开始工作前，给 Gemini CLI 发送这段话，让它进入状态：

> “现在切换到【数字分身模式】。请读取 `My_Digital_Avatar/00_Profile/profile.md` 和 `My_Digital_Avatar/03_Rules_Skills/system_rules.md`。获取当前上下文，准备执行任务。”

### 2. 场景演练：让它自主学习
**指令**：
> “作为分身，去 Google 搜一下‘2026 AI Agent 最佳实践’。结合我的 Profile，给我写一份简短的行动建议，存到 `01_Workspace` 里。记住，做完后要执行复利反思。”

**预期行为**：
1.  它会联网搜索。
2.  它会写文件。
3.  它会（如果不笨的话）在 `04_Logs` 里写一条：“今天学习了 Agent 实践，发现工具使用很重要。”
4.  它可能会在 `03_Rules_Skills` 里新建一条：“搜索技术类话题时，应该优先看 Github 和 arXiv。”

### 3. 场景演练：Vibe Coding (编写工具)
**指令**：
> “我觉得每次手动整理周报很烦。帮我写一个 Python 脚本，读取 `04_Logs` 里过去 7 天的文件，自动生成一份周报摘要。不要只给我代码，直接把脚本写到 `01_Workspace/weekly_report_gen.py` 并尝试运行它。”

---

## 🛡 维护与避坑指南

1.  **不要手动乱改文件**：尽量通过指令让 Gemini 改文件，这样它才知道发生了什么，保持“上下文一致性”。
2.  **严厉的爱**：如果它偷懒没写日志，或者重复犯错，要严厉批评它，并强制它更新 `mistakes.md`。这就是“养蛊”。
3.  **保持 Profile 简洁**：不要把所有琐事都塞进 Profile，把具体的做事流程拆分成 SOP 放到 `03_Rules_Skills` 里。
