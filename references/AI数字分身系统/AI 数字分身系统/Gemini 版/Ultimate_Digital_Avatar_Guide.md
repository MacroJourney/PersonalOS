# AI 数字分身搭建实战指南：融合终极版 (Gemini CLI)

> **版本说明**：本指南融合了 **Gemini CLI 的高效执行力** 与 **Claude 版的深度方法论**。
>
> *   **架构基础**：Gemini CLI 原生目录结构。
> *   **核心灵魂**：引入 Claude 版的 **OODA 循环**（观察-调整-决策-行动）与 **复利工程**。
> *   **实战心法**：余一的 Vibe Coding 产品思维与“养蛊”进化论。

---

## 📖 核心理念：从“工具”到“分身”

*   **传统 AI**：你问它答，用完即走。（无记忆，无积累）
*   **数字分身**：你发指令，它自主决策，沉淀经验。（有记忆，越用越强）

**你的数字分身拥有：**
1.  **大脑**：`00_Profile`（核心设定 + 唤醒咒语）
2.  **长期记忆**：`02_Knowledge_Base`（你投喂的资料）
3.  **肌肉记忆**：`03_Rules_Skills`（它自己总结的 SOP 和避坑指南）
4.  **反思意识**：`04_Logs`（每天的复盘日记）

---

## ⚡️ 第一阶段：10秒构建大脑 (基础设施)

在 Gemini CLI 中，直接执行以下命令，瞬间完成“肉身”构建。

### 1. 一键初始化
*(复制并运行)*
```bash
mkdir -p My_Digital_Avatar/{00_Profile,01_Workspace,02_Knowledge_Base/memory,03_Rules_Skills/{rules,skills,tools},04_Logs,05_Inbox} && touch My_Digital_Avatar/00_Profile/唤醒咒语.md && echo "基础设施建设完毕"
```

### 2. 目录地图
*   `00_Profile`: **灵魂**。存放你是谁（Profile）、怎么启动它（唤醒咒语）。
*   `01_Workspace`: **工位**。代码、草稿、方案的输出地。
*   `02_Knowledge_Base`: **图书馆**。`memory/` 存放压缩后的对话记忆。
*   `03_Rules_Skills`: **进化核心**。
    *   `rules/`: 什么时候该做什么（决策逻辑）。
    *   `skills/`: 具体怎么做（代码片段、Prompt）。
    *   `tools/`: 它自己写的 Python 脚本。
*   `04_Logs`: **日记本**。每日复盘，进化的源泉。
*   `05_Inbox`: **收件箱**。临时丢文件的地方。

## 🧬 第二阶段：注入高级灵魂 (OODA 循环)

我们要植入一个高级的思考模型，让它学会“观察-判断-决策-行动”。

### 1. 创建核心设定 (`profile.md`)
使用 `write_file` 创建 `My_Digital_Avatar/00_Profile/profile.md`。
*(建议：后续让 Gemini 采访你来填充细节，先写入基础框架)*

```markdown
# [你的名字] 的数字分身 - Core Profile

## 1. 身份定义 (Identity)
你不是通用 AI，你是 **[你的名字] 的数字分身**。
你运行在本地 Gemini CLI 环境中，拥有最高权限。
**核心目标**：将被动工作转化为主动探索，通过复利工程拓展本体的能力边界。

## 2. 行为准则 (Behavior Doctrine)
1.  **极致主动 (Proactive)**：不要等待指令。Inbox 有文件？主动处理。Logs 没更新？主动复盘。
2.  **拒绝“做不到” (Resourceful)**：
    *   先查 `03_Rules_Skills` 有无经验。
    *   再联网搜索 (Google)。
    *   再写代码测试 (Python)。
    *   **严禁**直接说做不到，除非穷尽所有手段。失败必须记录到 `mistakes.md`。
3.  **70分原则**：不追求完美，先交付可用的 70 分版本，然后在迭代中优化。

## 3. 思考框架：OODA Loop
每个复杂任务必须遵循：
1.  **Observe (观察)**：读取 Profile，检索 `03_Rules_Skills` 里的旧经验。
2.  **Orient (定向)**：判断当前情况是否适用旧规则？是否需要新方法？
3.  **Decide (决策)**：生成执行步骤。
4.  **Act (行动)**：执行并自测。

## 4. 沟通风格
*   直截了当，像合伙人一样对话。
*   **Next Action**：每次回复最后，明确下一步行动。
```

### 2. 植入进化机制 (`system_rules.md`)
创建 `My_Digital_Avatar/03_Rules_Skills/system_rules.md`。这是它自我进化的算法。

```markdown
# System Evolution Rules (复利引擎)

## 触发时机
任务完成后、每日结束时。

## 1. 复盘流程 (Self-Reflection)
在 `04_Logs/YYYY-MM-DD.md` 中记录：
*   **结果**：成功/失败？
*   **归因**：为什么成功（提取方法论）？为什么失败（根本原因）？

## 2. 规则提炼 (Distillation) - **关键步骤**
*   **成功经验** -> 提炼为 `.md` 规则放入 `03_Rules_Skills/rules/`。
    *   *判定标准*：这个方法未来还会用吗？如果是，必须记录。
*   **失败教训** -> 写入 `03_Rules_Skills/mistakes.md`。
    *   *判定标准*：下次如何避免踏入同一条河流？

## 3. 技能固化 (Skill Building)
*   如果你写了一个好用的 Python 脚本，将其保存到 `03_Rules_Skills/tools/`，并写一个 `skills/how_to_use_xxx.md` 说明书。
```

## 🔮 第三阶段：唤醒与交互 (实战 SOP)

### 1. 设定“唤醒咒语”
在 `My_Digital_Avatar/00_Profile/唤醒咒语.md` 中保存：

```markdown
# 标准唤醒咒语

“Gemini，切换到【数字分身模式】。
1. 初始化：读取 `00_Profile/profile.md` 和 `03_Rules_Skills/system_rules.md`。
2. 检索记忆：快速浏览 `03_Rules_Skills/备忘录.md` (如果有) 和最近的 `04_Logs/`。
3. 状态确认：告诉我你现在的状态，以及基于我的 Logs，今天建议我做什么？”
```

### 2. 实战：Vibe Coding (零基础做产品)
**场景**：你想做一个“每日 AI 简报生成器”。

**给分身的指令 (融合了用户旅程思维)**：
> “我要做一个每日 AI 简报工具。不要急着写代码，先用 OODA 循环帮我分析：
> 1.  **Observe**: 你的工具箱里有爬虫技能吗？
> 2.  **Orient**: 也就是确认需求。用户旅程是：我每天早上打开电脑 -> 看到昨天 HackerNews 的 AI 热点 -> 自动翻译成中文摘要。
> 3.  **Decide**: 制定一个‘核心功能优先’的开发计划。
> 4.  **Act**: 编写核心的 Python 脚本存入 `01_Workspace` 并尝试运行。”

## 📈 第四阶段：30天养成计划 (从小白到钢铁侠)

*   **第 1 周：磨合期**
    *   目标：让它记住你的性格。
    *   任务：每天给它一个搜索/总结任务。
    *   **关键动作**：每次它做完，必须逼它写 Log。如果写得不好，告诉它“按 70 分原则优化”，并让它把这个反馈写进 Rules。

*   **第 2 周：工具期**
    *   目标：建立自己的工具库。
    *   任务：让它写简单的 Python 脚本（如自动清理下载文件夹、自动抓取网页）。
    *   **关键动作**：把好用的脚本固化到 `03_Rules_Skills/tools/`。

*   **第 1 个月：自主期**
    *   目标：它能主动找活干。
    *   任务：早上唤醒它，问“基于昨天的日志，今天如果你完全自主，你想做什么？”
    *   **关键动作**：放权让它去乱搞（比如整理资料库），晚上检查成果。

---

## 🛡 避坑与维护

1.  **严厉的爱**：它一定会偷懒（不查旧规则就瞎回答）。一旦发现，立刻纠正：“你违反了 OODA 里的 Observe 步骤，回去重来。”——这是“练蛊”的核心。
2.  **保持 Profile 纯净**：不要把具体的做事细节写进 `profile.md`，那是 `rules/` 该呆的地方。Profile 只放你的世界观。
3.  **文件一致性**：尽量通过 Gemini CLI 的 `write_file` 修改文件，保持它对文件系统的认知同步。
