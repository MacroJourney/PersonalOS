# /review-pos — PersonalOS 文档审查技能

对照 `docs/00_核心设计.md` 审查 PersonalOS 项目文档，确保与 v2.1 架构对齐。

## 参数

- `$1`：目标文档路径（待审查）。如未提供，列出 `docs/` 下所有文档供用户选择

## 步骤

1. 读取 `docs/00_核心设计.md`，提取架构要素：四区工作空间、三层 PDCA、AI 角色、复利机制
2. 读取 `CLAUDE.md` 中的术语表和写作规范
3. 读取目标文档，逐节对比，输出差距分析表格：
   - **位置**：目标文档中的章节
   - **类型**：缺失 / 术语不一致 / 路径错误 / 结构偏差 / 冗余 / 写作规范违规
   - **详情**：具体问题描述
   - **建议**：修改方案
4. 等待用户确认后再执行修改
5. 每次只修改一个章节，修改后等待确认
6. 全部修改完成后，检查是否需要同步更新其他文档的交叉引用

## 检查清单

- [ ] 术语与核心设计一致（参照 CLAUDE.md 术语表）
- [ ] 路径引用使用相对路径，包含 `.md` 扩展名
- [ ] 列表用 `-`，引号用「」，强调用 `**粗体**`
- [ ] 中英文之间有空格
- [ ] 不引入与核心设计矛盾的概念
- [ ] 缺失内容明确标记为缺失，不合理化

## 注意事项

- 所有输出使用中文
- 不对系统架构做主观假设（如模块用途、Vault 划分、工具限制）
- 不确定时先问用户，不要自行推测
